# BDD 流程視覺化圖表

## 完整 BDD 流程圖

```
┌─────────────────────────────────────────────────────────────────┐
│                        BDD 開發循環                              │
└─────────────────────────────────────────────────────────────────┘

   ┌──────────────┐
   │  1. 推測 🤔  │  識別業務價值和假設
   │  Speculate   │  定義成功標準
   └──────┬───────┘
          │
          ↓
   ┌──────────────┐
   │  2. 說明 📖  │  Example Mapping 工作坊
   │  Illustrate  │  創建具體範例
   └──────┬───────┘
          │
          ↓
   ┌──────────────┐
   │  3. 制定 ✍️  │  編寫 Gherkin 場景
   │  Formulate   │  組織 .feature 文件
   └──────┬───────┘
          │
          ↓
   ┌──────────────┐
   │  4. 自動化 🤖│  實現步驟定義
   │  Automate    │  開發業務邏輯
   └──────┬───────┘
          │
          ↓
   ┌──────────────┐
   │  5. 展示 📊  │  執行測試
   │  Demonstrate │  生成報告
   └──────┬───────┘
          │
          ↓
   ┌──────────────┐
   │  6. 驗證 ✅  │  業務驗收
   │  Validate    │  度量指標
   └──────┬───────┘
          │
          ↓
    ┌─────────┐
    │ 成功？   │
    └────┬────┘
         │
    ┌────┴────┐
    │         │
   是        否
    │         │
    ↓         ↓
  發布    改進並重新推測
    │         │
    └─────────┘
```

## Example Mapping 流程圖

```
Example Mapping 工作坊
═══════════════════════════════════════════════════════════════

準備階段 (5 分鐘)
┌─────────────────────────────────────────────────────┐
│ • 準備便利貼（黃、藍、綠、紅）                        │
│ • 邀請 Three Amigos                                  │
│ • 選擇一個使用者故事                                 │
└─────────────────────────────────────────────────────┘
                         ↓

探索規則 (10 分鐘)
┌─────────────────────────────────────────────────────┐
│  📋 使用者故事（黃色）                                │
│  ┌─────────────────────────────────────────┐        │
│  │ 作為 [角色]                              │        │
│  │ 我想要 [功能]                            │        │
│  │ 以便 [價值]                              │        │
│  └─────────────────────────────────────────┘        │
│                                                      │
│  ↓ 識別規則                                         │
│                                                      │
│  📏 規則 1 (藍色)    📏 規則 2 (藍色)                │
└─────────────────────────────────────────────────────┘
                         ↓

創建範例 (15 分鐘)
┌─────────────────────────────────────────────────────┐
│  📏 規則 1                                           │
│    ├─ 📝 範例 1.1 (綠色)                            │
│    ├─ 📝 範例 1.2 (綠色)                            │
│    └─ 📝 範例 1.3 (綠色)                            │
│                                                      │
│  📏 規則 2                                           │
│    ├─ 📝 範例 2.1 (綠色)                            │
│    └─ 📝 範例 2.2 (綠色)                            │
└─────────────────────────────────────────────────────┘
                         ↓

識別問題 (5 分鐘)
┌─────────────────────────────────────────────────────┐
│  ❓ 限額是否因帳戶類型而異？(紅色)                    │
│  ❓ 需要雙重認證嗎？(紅色)                           │
│  ❓ 轉帳是即時的嗎？(紅色)                           │
└─────────────────────────────────────────────────────┘
                         ↓

決策
┌─────────────────────────────────────────────────────┐
│  問題 < 3 個？                                       │
│  └─ 是 → 準備好制定場景                             │
│  └─ 否 → 需要更多研究或拆分故事                      │
└─────────────────────────────────────────────────────┘
```

## Three Amigos 協作流程

```
Three Amigos 會議
═══════════════════════════════════════════════════════

   ┌──────────────┐
   │   業務代表   │  定義「為什麼」和「什麼」
   │  🎯 PO/BA    │  → 業務價值、規則、優先級
   └───────┬──────┘
           │
           ↓
   ┌──────────────┐
   │   開發人員   │  探索「如何」和可行性
   │  💻 Dev      │  → 技術方案、估算、風險
   └───────┬──────┘
           │
           ↓
   ┌──────────────┐
   │   測試人員   │  關注「是否」和邊界
   │  🧪 QA       │  → 測試場景、邊界案例、驗收標準
   └───────┬──────┘
           │
           ↓
    ┌─────────────┐
    │  共同理解    │
    │  📝 輸出：   │
    │  • 範例集合  │
    │  • 業務規則  │
    │  • 問題清單  │
    └─────────────┘


會議時間軸
═══════════════════════════════════════════════════════

0-5 分鐘      │ 設定背景和目標
              │ • 閱讀使用者故事
              │ • 確認參與者理解
              │
5-15 分鐘     │ 探索業務規則
              │ • 「在什麼情況下...」
              │ • 「規則是否總是適用？」
              │ • 「有例外嗎？」
              │
15-25 分鐘    │ 創建具體範例
              │ • 「給我一個例子...」
              │ • 「如果...會怎樣？」
              │ • 「邊界在哪裡？」
              │
25-30 分鐘    │ 總結和行動
              │ • 確認理解一致
              │ • 分配問題責任
              │ • 決定是否準備好開發
```

## Gherkin 場景結構

```
Feature 文件結構
═══════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────┐
│ Feature: 功能名稱                                    │
│   功能描述和業務價值                                 │
│                                                      │
│ ┌─────────────────────────────────────────────────┐│
│ │ Background: 共同前置條件                        ││
│ │   Given 所有場景共用的設置                      ││
│ │   And 更多共同設置                              ││
│ └─────────────────────────────────────────────────┘│
│                                                      │
│ ┌─────────────────────────────────────────────────┐│
│ │ @tag1 @tag2                                     ││
│ │ Scenario: 場景 1                                ││
│ │   Given 前置條件                                ││
│ │   When 執行動作                                 ││
│ │   Then 預期結果                                 ││
│ └─────────────────────────────────────────────────┘│
│                                                      │
│ ┌─────────────────────────────────────────────────┐│
│ │ Scenario Outline: 場景範本                      ││
│ │   Given <參數1> 的前置條件                      ││
│ │   When 執行 <參數2>                             ││
│ │   Then 得到 <結果>                              ││
│ │                                                 ││
│ │   Examples:                                     ││
│ │     | 參數1 | 參數2 | 結果 |                    ││
│ │     | 值1   | 值2   | 值3  |                    ││
│ │     | 值4   | 值5   | 值6  |                    ││
│ └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘


Given-When-Then 模式
═══════════════════════════════════════════════════════

┌─────────────────┐
│  Given (給定)    │  設置測試的前置條件
│  ────────────   │  • 建立初始狀態
│  • 測試數據     │  • 配置系統
│  • 系統狀態     │  • 準備環境
│  • 使用者角色   │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  When (當)       │  執行被測試的動作
│  ────────────   │  • 使用者操作
│  • 業務操作     │  • 系統事件
│  • API 調用     │  • 外部觸發
│  • 事件觸發     │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Then (那麼)     │  驗證預期結果
│  ────────────   │  • 檢查狀態變化
│  • 系統回應     │  • 驗證輸出
│  • 狀態檢查     │  • 確認副作用
│  • 錯誤訊息     │
└─────────────────┘
```

```
═══════════════════════════════════════════════════════

     │

     │
```

## 測試執行流程

```
測試執行生命週期
═══════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────┐
│ TestMain()                                           │
│   • 解析命令列參數                                   │
│   • 設置測試路徑                                     │
│   • 初始化全局配置                                   │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓
┌─────────────────────────────────────────────────────┐
│ BeforeSuite Hook                                     │
│   • 設置測試環境                                     │
│   • 初始化資料庫                                     │
│   • 啟動測試服務                                     │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓ 對每個 Scenario
┌─────────────────────────────────────────────────────┐
│ BeforeScenario Hook                                  │
│   • 創建新的 TestContext                             │
│   • 重置測試數據                                     │
│   • 準備場景環境                                     │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓ 執行場景步驟
┌─────────────────────────────────────────────────────┐
│ Step Execution                                       │
│   Given → When → Then                                │
│     ↓      ↓      ↓                                  │
│   成功    成功    成功  → Scenario PASSED            │
│                   失敗  → Scenario FAILED            │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓
┌─────────────────────────────────────────────────────┐
│ AfterScenario Hook                                   │
│   • 清理測試數據                                     │
│   • 重置 TestContext                                 │
│   • 記錄執行結果                                     │
└──────────────────┬──────────────────────────────────┘
                   │
                   ↓ 重複直到所有場景完成
┌─────────────────────────────────────────────────────┐
│ AfterSuite Hook                                      │
│   • 關閉測試服務                                     │
│   • 清理資源                                         │
│   • 生成測試報告                                     │
└─────────────────────────────────────────────────────┘
```

## CI/CD 整合流程

```
持續整合流程
═══════════════════════════════════════════════════════

┌─────────────┐
│ Git Push    │
└──────┬──────┘
       │
       ↓
┌─────────────────────────────────────────────────────┐
│ CI Server (GitHub Actions / Jenkins)                 │
│                                                      │
│  ┌────────────────────────────────────────────────┐│
│  │ 1. Checkout Code                               ││
│  │    git clone / git pull                        ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│  ┌────────────────────────────────────────────────┐│
│  │ 2. Install Dependencies                        ││
│  │    go mod download                             ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│  ┌────────────────────────────────────────────────┐│
│  │ 3. Run Unit Tests                              ││
│  │    go test ./...                               ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│  ┌────────────────────────────────────────────────┐│
│  │ 4. Run BDD Tests                               ││
│  │    make test                                   ││
│  │    - Smoke tests (@smoke)                      ││
│  │    - Full test suite                           ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│  ┌────────────────────────────────────────────────┐│
│  │ 5. Generate Reports                            ││
│  │    make test-cucumber-report                   ││
│  │    - Cucumber JSON                             ││
│  │    - Coverage HTML                             ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│  ┌────────────────────────────────────────────────┐│
│  │ 6. Quality Gates                               ││
│  │    ✓ Test pass rate > 95%                     ││
│  │    ✓ Coverage > 80%                            ││
│  │    ✓ No critical bugs                          ││
│  └────────────────────────────────────────────────┘│
│                  ↓                                   │
│           ┌──────┴───────┐                          │
│           │              │                          │
│       成功│              │失敗                      │
│           ↓              ↓                          │
│     ┌─────────┐    ┌─────────┐                     │
│     │ Deploy  │    │ Notify  │                     │
│     │ to Env  │    │ Team    │                     │
│     └─────────┘    └─────────┘                     │
└─────────────────────────────────────────────────────┘


報告發佈
═══════════════════════════════════════════════════════

     │
     ├─→ Slack 通知
     │     • 測試摘要
     │     • 失敗場景
     │     • 報告連結
     │
     ├─→ Email 報告
     │     • 詳細結果
     │     • 趨勢分析
     │
     ├─→ Dashboard
     │     • 即時狀態
     │     • 歷史趨勢
     │     • 覆蓋率圖表
     │
     └─→ JIRA / 問題追蹤
           • 自動建立缺陷
           • 更新測試狀態
```

## 反饋循環

```
持續改進循環
═══════════════════════════════════════════════════════

     ┌──────────────────────────────────────┐
     │         執行測試                      │
     │    Run BDD Test Suite                │
     └──────────────┬───────────────────────┘
                    │
                    ↓
     ┌──────────────────────────────────────┐
     │         收集數據                      │
     │  • 通過率                            │
     │  • 執行時間                          │
     │  • 失敗原因                          │
     │  • 覆蓋率                            │
     └──────────────┬───────────────────────┘
                    │
                    ↓
     ┌──────────────────────────────────────┐
     │         分析結果                      │
     │  • 識別模式                          │
     │  • 發現瓶頸                          │
     │  • 評估風險                          │
     └──────────────┬───────────────────────┘
                    │
                    ↓
     ┌──────────────────────────────────────┐
     │         採取行動                      │
     │  • 重構場景                          │
     │  • 優化測試                          │
     │  • 更新文檔                          │
     │  • 培訓團隊                          │
     └──────────────┬───────────────────────┘
                    │
                    ↓
     ┌──────────────────────────────────────┐
     │         驗證改進                      │
     │  • 對比指標                          │
     │  • 確認效果                          │
     │  • 調整策略                          │
     └──────────────┬───────────────────────┘
                    │
                    ↓
                回到執行測試

每週改進循環
═══════════════════════════════════════════════════════

週一 │ 回顧上週測試結果
     │ 識別問題和改進點
     │
週二 │ 重構測試場景
     │ 更新步驟定義
     │
週三 │ 執行並驗證改進
     │ 收集新的數據
     │
週四 │ 分析趨勢
     │ 調整策略
     │
週五 │ 團隊分享
     │ 記錄學習
```
